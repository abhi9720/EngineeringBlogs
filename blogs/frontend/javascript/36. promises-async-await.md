# Promises and Async/Await: Complete Guide

## Overview

Promises are the foundation of asynchronous programming in modern JavaScript. They provide a cleaner way to handle async operations compared to callback-based code, and async/await makes asynchronous code look and behave like synchronous code.

Understanding Promises deeply is essential for building reliable asynchronous applications.

---

## How Promises Work Internally

### Promise States

A Promise can be in one of three states:

```
┌─────────────────────────────────────────────────────────────────┐
│                      Promise States                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌──────────┐                                                 │
│   │ Pending  │ ──────────────────► Fulfilled                   │
│   │          │      resolve()        │                         │
│   └──────────┘                       │                         │
│         │                            │                         │
│         │ (initial state)            ▼                         │
│         │                     ┌───────────┐                   │
│         │                     │ Fulfilled │                   │
│         │                     │ (resolved)│                   │
│         └────────────────────►│           │                   │
│              reject()         └───────────┘                   │
│                                                                  │
│   States can only transition: pending → fulfilled/rejected     │
│   Once settled, promise is immutable                           │
└─────────────────────────────────────────────────────────────────┘
```

### Promise Lifecycle

```javascript
// Creating a promise
const promise = new Promise((resolve, reject) => {
    // This executor runs immediately

    setTimeout(() => {
        const data = fetchData();

        if (data) {
            resolve(data);  // Transition to fulfilled
        } else {
            reject(new Error('No data'));  // Transition to rejected
        }
    }, 1000);
});

// Promise states
console.log(promise);  // Promise { <pending> }

// After resolution
promise
    .then(data => console.log(data))  // Runs when fulfilled
    .catch(error => console.error(error));  // Runs when rejected
```

### Promise Chain

```javascript
// Promises can be chained - each .then() returns a new promise
fetch('/api/user')
    .then(response => response.json())  // Returns promise
    .then(user => user.name)  // Returns string
    .then(name => console.log(name))  // Logs the name
    .catch(error => console.error(error));  // Catches any error

// Chain flow:
// fetch() returns P1
// P1.then() returns P2 (from response.json())
// P2.then() returns P3 (from user.name)
// Each .then() waits for previous promise to resolve
```

## Real-World JavaScript Use Cases

### Case 1: Sequential Async Operations

```javascript
// Problem: Async operations that depend on each other
async function createUserAndSendWelcome(email) {
    // Must wait for each step
    const user = await createUser(email);
    await sendWelcomeEmail(user.id);
    await logAnalytics('user.created', { userId: user.id });

    return user;
}

// Equivalent with .then()
function createUserAndSendWelcomeThen(email) {
    return createUser(email)
        .then(user =>
            sendWelcomeEmail(user.id)
                .then(() => user)
        )
        .then(user =>
            logAnalytics('user.created', { userId: user.id })
                .then(() => user)
        );
}
```

### Case 2: Parallel Async Operations

```javascript
// When operations don't depend on each other, run in parallel
async function loadDashboardData() {
    // These can all run simultaneously
    const [user, orders, notifications] = await Promise.all([
        fetchUser(),
        fetchOrders(),
        fetchNotifications()
    ]);

    return { user, orders, notifications };
}

// With timeout - race against time
async function fetchWithTimeout(url, timeout = 5000) {
    const controller = new AbortController();
    const timeoutId = setTimeout(() => controller.abort(), timeout);

    try {
        const response = await fetch(url, { signal: controller.signal });
        return response;
    } finally {
        clearTimeout(timeoutId);
    }
}

// Promise.allSettled - get results regardless of individual failures
async function fetchAllProducts() {
    const results = await Promise.allSettled([
        fetchProduct(1),
        fetchProduct(2),
        fetchProduct(3)
    ]);

    return results.map((result, index) => ({
        id: index + 1,
        status: result.status,
        data: result.status === 'fulfilled' ? result.value : null,
        error: result.status === 'rejected' ? result.reason : null
    }));
}
```

### Case 3: Error Handling Patterns

```javascript
// Try-catch with async/await
async function safeFetch(url) {
    try {
        const response = await fetch(url);

        if (!response.ok) {
            throw new Error(`HTTP ${response.status}`);
        }

        return await response.json();

    } catch (error) {
        if (error.name === 'AbortError') {
            return { error: 'Request timed out' };
        }
        if (error.name === 'TypeError') {
            return { error: 'Network error' };
        }
        throw error;  // Re-throw unknown errors
    }
}

// Error boundaries for multiple operations
async function processMultiple(items) {
    const results = [];

    for (const item of items) {
        try {
            const result = await processItem(item);
            results.push({ success: true, data: result });
        } catch (error) {
            results.push({ success: false, error: error.message });
        }
    }

    return results;
}

// Retry logic with exponential backoff
async function fetchWithRetry(url, maxRetries = 3) {
    for (let attempt = 1; attempt <= maxRetries; attempt++) {
        try {
            return await fetch(url);
        } catch (error) {
            if (attempt === maxRetries) throw error;

            const delay = Math.pow(2, attempt) * 100;
            console.log(`Attempt ${attempt} failed, retrying in ${delay}ms...`);
            await new Promise(resolve => setTimeout(resolve, delay));
        }
    }
}
```

### Case 4: Concurrent Limit (Semaphore Pattern)

```javascript
// Limit concurrent async operations
class AsyncQueue {
    constructor(concurrency) {
        this.concurrency = concurrency;
        this.running = 0;
        this.queue = [];
    }

    async add(promiseFn) {
        return new Promise((resolve, reject) => {
            this.queue.push({ fn: promiseFn, resolve, reject });
            this.process();
        });
    }

    async process() {
        if (this.running >= this.concurrency || !this.queue.length) {
            return;
        }

        this.running++;
        const { fn, resolve, reject } = this.queue.shift();

        try {
            const result = await fn();
            resolve(result);
        } catch (error) {
            reject(error);
        } finally {
            this.running--;
            this.process();
        }
    }
}

// Usage
const queue = new AsyncQueue(3);  // Max 3 concurrent

const results = await Promise.all([
    queue.add(() => fetch('/api/1').then(r => r.json())),
    queue.add(() => fetch('/api/2').then(r => r.json())),
    queue.add(() => fetch('/api/3').then(r => r.json())),
    queue.add(() => fetch('/api/4').then(r => r.json())),
    queue.add(() => fetch('/api/5').then(r => r.json()))
]);
// First 3 run, then 4th starts when one completes
```

### Case 5: Promise Composition Utilities

```javascript
// Promise.all - wait for all to complete (fails fast)
await Promise.all([
    saveToDatabase(data1),
    saveToDatabase(data2),
    saveToDatabase(data3)
]);

// Promise.race - first one to settle wins
async function fetchWithTimeout(url, timeout) {
    return Promise.race([
        fetch(url),
        new Promise((_, reject) =>
            setTimeout(() => reject(new Error('Timeout')), timeout)
        )
    ]);
}

// Promise.any - first to succeed (ignores rejections)
async function fetchFirst(urls) {
    return Promise.any(urls.map(url => fetch(url)));
}

// Promise.allSettled - wait for all to settle
const results = await Promise.allSettled([
    fetch('/api/a'),
    fetch('/api/b'),
    fetch('/api/c')
]);

results.forEach((result, index) => {
    if (result.status === 'fulfilled') {
        console.log(`Request ${index} succeeded:`, result.value);
    } else {
        console.log(`Request ${index} failed:`, result.reason);
    }
});
```

## Trade-offs: Promise vs Callback vs Async/Await

### Comparison

| Approach | Pros | Cons |
|----------|------|------|
| Callbacks | Simple, no dependencies | Callback hell, hard to handle errors |
| Promises | Chainable, better error handling | Still verbose for complex flows |
| Async/Await | Syntactic sugar, readable | Can mask async behavior |

### When to Use Each

```javascript
// Callback: Simple async with single completion
element.addEventListener('click', (event) => {
    console.log('Clicked!');
});

// Promises: Multiple async operations with error handling
fetch(url)
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error(error));

// Async/await: Complex async logic that reads sequentially
async function getData() {
    try {
        const response = await fetch(url);
        const data = await response.json();
        return data;
    } catch (error) {
        console.error(error);
    }
}
```

## Production Considerations

### 1. Avoiding Unhandled Promise Rejections

```javascript
// Always handle rejections
fetch('/api/data')
    .then(data => process(data))
    .catch(error => {
        console.error('Fetch failed:', error);
        // Don't leave promises unhandled!
    });

// Global handler for uncaught rejections
window.addEventListener('unhandledrejection', (event) => {
    console.error('Unhandled rejection:', event.reason);
    // Report to error tracking service
    reportError(event.reason);
});

// In Node.js
process.on('unhandledRejection', (reason, promise) => {
    console.error('Unhandled Rejection at:', promise, 'reason:', reason);
});
```

### 2. Promise Performance

```javascript
// Don't create unnecessary promises
// WRONG
async function badExample(items) {
    return Promise.all(items.map(item =>
        new Promise((resolve) => {
            processItem(item, result => resolve(result));
        })
    ));
}

// CORRECT - use Promise constructor only when needed
function goodExample(items) {
    return Promise.all(items.map(item =>
        new Promise(resolve => processItem(item, resolve))
    ));
}

// Even better - existing promise
function betterExample(items) {
    return Promise.all(items.map(item => processItemAsync(item)));
}
```

### 3. Cancellation with AbortController

```javascript
// Modern way to cancel async operations
async function fetchWithCancellation() {
    const controller = new AbortController();
    const signal = controller.signal;

    try {
        const response = await fetch('/api/large-data', { signal });

        // Cancel after 5 seconds
        setTimeout(() => controller.abort(), 5000);

        return await response.json();
    } catch (error) {
        if (error.name === 'AbortError') {
            console.log('Request cancelled');
            return null;
        }
        throw error;
    }
}

// Cancel multiple requests
async function fetchAllWithCancellation(urls) {
    const controller = new AbortController();
    const promises = urls.map(url =>
        fetch(url, { signal: controller.signal }).then(r => r.json())
    );

    // Cancel all after 3 seconds
    setTimeout(() => controller.abort(), 3000);

    return Promise.all(promises).catch(error => {
        if (error.name === 'AbortError') {
            return [];
        }
        throw error;
    });
}
```

## Common Mistakes

### Mistake 1: Forgetting to Await

```javascript
// WRONG: Not awaiting causes race conditions
async function badPattern() {
    fetch('/api/save', { method: 'POST', body: JSON.stringify({ a: 1 }) });
    // fetch starts but doesn't wait - function may exit before completes!

    // The request might be cancelled before completion
}

// CORRECT: Always await promises
async function goodPattern() {
    await fetch('/api/save', { method: 'POST', body: JSON.stringify({ a: 1 }) });
    console.log('Saved!');
}

// Or if not awaiting, at least handle errors
async function acceptablePattern() {
    fetch('/api/save', { method: 'POST', body: JSON.stringify({ a: 1 }) })
        .catch(error => console.error(error));
}
```

### Mistake 2: Not Handling Errors

```javascript
// WRONG: Missing catch causes unhandled rejection
async function badErrorHandling() {
    const response = await fetch('/api/nonexistent');
    return response.json();
}

// CORRECT: Always handle errors
async function goodErrorHandling() {
    try {
        const response = await fetch('/api/nonexistent');

        if (!response.ok) {
            throw new Error(`HTTP ${response.status}`);
        }

        return await response.json();
    } catch (error) {
        console.error('Fetch failed:', error);
        return null;
    }
}
```

### Mistake 3: Using forEach with Async

```javascript
// WRONG: forEach doesn't wait for async functions
async function badPattern(items) {
    items.forEach(async (item) => {
        await processItem(item);  // Won't wait!
    });

    console.log('All done');  // Runs before items are processed!
}

// CORRECT: Use for...of or map with Promise.all
async function goodPatternForOf(items) {
    for (const item of items) {
        await processItem(item);  // Sequential
    }
    console.log('All done');
}

async function goodPatternMap(items) {
    await Promise.all(items.map(item => processItem(item)));
    console.log('All done');  // Parallel execution
}
```

### Mistake 4: Not Returning in .then()

```javascript
// WRONG: Forgetting to return breaks the chain
async function badChain() {
    fetch('/api/data')
        .then(response => response.json())
        .then(data => {
            console.log(data);
            // Missing return!
        })
        .then(result => console.log(result));  // undefined!
}

// CORRECT: Always return values to pass through chain
async function goodChain() {
    fetch('/api/data')
        .then(response => response.json())
        .then(data => {
            console.log(data);
            return data;  // Pass to next .then()
        })
        .then(data => console.log(data));  // Has the data!
}
```

### Mistake 5: Mixing .then and Async/Await

```javascript
// Mixing can cause confusion
async function mixedPattern() {
    // Using .then inside async/await
    const data = await fetch('/api/data')
        .then(response => response.json());

    // This works but is inconsistent
    return data;
}

// Choose one style and stick with it
// Pure async/await
async function pureAsyncAwait() {
    const response = await fetch('/api/data');
    const data = await response.json();
    return data;
}

// Pure Promise chain
function purePromiseChain() {
    return fetch('/api/data')
        .then(response => response.json())
        .then(data => data);
}
```

## Summary

Promises and async/await are essential for modern JavaScript:

1. **Promise States**: pending → fulfilled/rejected (immutable once settled)
2. **Promise.all**: Parallel execution, fails fast
3. **Promise.allSettled**: Get results regardless of individual failures
4. **Async/Await**: Syntactic sugar that makes async code synchronous-looking

Key takeaways:
- Always handle promise rejections (use .catch or try/catch)
- Use Promise.all for parallel operations when possible
- Use AbortController for cancellation
- Prefer async/await for readability, but understand the underlying promises

Master these patterns to write clean, maintainable asynchronous JavaScript.
