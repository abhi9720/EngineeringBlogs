# Encryption at Rest & in Transit

## Overview

Encryption protects data in two states: in transit (moving across networks) and at rest (stored on disk or in databases). TLS/SSL secures communication between services, while encryption at rest protects against physical data theft. This guide covers the handshake protocols, key management strategies, and production-grade patterns for implementing encryption in distributed systems.

## TLS 1.3 Handshake

TLS 1.3 establishes an encrypted channel between client and server with a 1-RTT handshake (or 0-RTT for resumed connections).

## mTLS (Mutual TLS)

In mTLS, both parties present certificates to authenticate each other, essential for zero-trust service-to-service communication.

```java
@Configuration
public class MutualTlsConfig {
    
    @Bean
    public SSLContext mtlsContext() throws Exception {
        // Load client certificate and private key
        KeyStore keyStore = KeyStore.getInstance("PKCS12");
        try (InputStream is = new FileInputStream("client.p12")) {
            keyStore.load(is, "keystore-password".toCharArray());
        }
        
        KeyManagerFactory kmf = KeyManagerFactory.getInstance("SunX509");
        kmf.init(keyStore, "key-password".toCharArray());
        
        // Load trusted CA certificates
        KeyStore trustStore = KeyStore.getInstance("PKCS12");
        try (InputStream is = new FileInputStream("truststore.p12")) {
            trustStore.load(is, "truststore-password".toCharArray());
        }
        
        TrustManagerFactory tmf = TrustManagerFactory.getInstance("SunX509");
        tmf.init(trustStore);
        
        SSLContext sslContext = SSLContext.getInstance("TLSv1.3");
        sslContext.init(kmf.getKeyManagers(), tmf.getTrustManagers(), null);
        
        return sslContext;
    }
    
    @Bean
    public RestTemplate mtlsRestTemplate() throws Exception {
        CloseableHttpClient httpClient = HttpClients.custom()
            .setSSLContext(mtlsContext())
            .setSSLHostnameVerifier(NoopHostnameVerifier.INSTANCE)
            .build();
        
        HttpComponentsClientHttpRequestFactory factory = 
            new HttpComponentsClientHttpRequestFactory(httpClient);
        
        return new RestTemplate(factory);
    }
}
```

## Symmetric vs Asymmetric Encryption

```java
public class EncryptionDemo {
    private static final String ALGORITHM = "AES/GCM/NoPadding";
    
    // Symmetric Encryption (AES-256-GCM)
    public byte[] encryptSymmetric(byte[] plaintext, SecretKey key) throws Exception {
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        byte[] iv = new byte[12]; // 96-bit IV for GCM
        SecureRandom.getInstanceStrong().nextBytes(iv);
        
        GCMParameterSpec spec = new GCMParameterSpec(128, iv); // 128-bit tag
        cipher.init(Cipher.ENCRYPT_MODE, key, spec);
        
        byte[] ciphertext = cipher.doFinal(plaintext);
        
        // Prepend IV to ciphertext for storage
        byte[] encrypted = new byte[iv.length + ciphertext.length];
        System.arraycopy(iv, 0, encrypted, 0, iv.length);
        System.arraycopy(ciphertext, 0, encrypted, iv.length, ciphertext.length);
        
        return encrypted;
    }
    
    public byte[] decryptSymmetric(byte[] encrypted, SecretKey key) throws Exception {
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        
        // Extract IV from first 12 bytes
        byte[] iv = new byte[12];
        System.arraycopy(encrypted, 0, iv, 0, 12);
        
        GCMParameterSpec spec = new GCMParameterSpec(128, iv);
        cipher.init(Cipher.DECRYPT_MODE, key, spec);
        
        return cipher.doFinal(encrypted, 12, encrypted.length - 12);
    }
    
    // Asymmetric Encryption (RSA-OAEP)
    public byte[] encryptAsymmetric(byte[] data, PublicKey publicKey) throws Exception {
        Cipher cipher = Cipher.getInstance("RSA/ECB/OAEPWithSHA-256AndMGF1Padding");
        cipher.init(Cipher.ENCRYPT_MODE, publicKey);
        return cipher.doFinal(data);
    }
    
    // Hybrid Encryption: Encrypt AES key with RSA, then encrypt data with AES
    public EncryptedMessage hybridEncrypt(byte[] data, PublicKey publicKey) throws Exception {
        // Generate random AES key
        KeyGenerator keyGen = KeyGenerator.getInstance("AES");
        keyGen.init(256);
        SecretKey aesKey = keyGen.generateKey();
        
        // Encrypt AES key with RSA
        byte[] encryptedKey = encryptAsymmetric(aesKey.getEncoded(), publicKey);
        
        // Encrypt data with AES-GCM
        byte[] encryptedData = encryptSymmetric(data, aesKey);
        
        return new EncryptedMessage(encryptedKey, encryptedData);
    }
}
```

## AWS KMS Integration

```java
@Service
public class KmsEncryptionService {
    
    private final KmsClient kmsClient;
    private final String keyId;
    private final Map<String, DataKey> dataKeyCache = new ConcurrentHashMap<>();
    
    public KmsEncryptionService(KmsClient kmsClient, 
            @Value("${kms.key.id}") String keyId) {
        this.kmsClient = kmsClient;
        this.keyId = keyId;
    }
    
    public EncryptedData encrypt(String plaintext, String encryptionContext) {
        // Generate a data key (envelope encryption)
        GenerateDataKeyResponse dataKeyResponse = kmsClient.generateDataKey(
            GenerateDataKeyRequest.builder()
                .keyId(keyId)
                .keySpec("AES_256")
                .encryptionContext(Map.of("purpose", encryptionContext))
                .build()
        );
        
        // Encrypt with the plaintext data key
        byte[] ciphertext = aesEncrypt(
            plaintext.getBytes(StandardCharsets.UTF_8),
            dataKeyResponse.plaintext().asByteArray()
        );
        
        // Store only the encrypted data key (discard plaintext key)
        return EncryptedData.builder()
            .ciphertext(ciphertext)
            .encryptedDataKey(dataKeyResponse.ciphertextBlob().asByteArray())
            .encryptionContext(encryptionContext)
            .build();
    }
    
    public String decrypt(EncryptedData encryptedData) {
        // Decrypt the data key using KMS
        DecryptResponse decryptResponse = kmsClient.decrypt(
            DecryptRequest.builder()
                .ciphertextBlob(SdkBytes.fromByteArray(
                    encryptedData.getEncryptedDataKey()))
                .encryptionContext(Map.of("purpose", 
                    encryptedData.getEncryptionContext()))
                .build()
        );
        
        // Decrypt the data with the plaintext key
        byte[] plaintext = aesDecrypt(
            encryptedData.getCiphertext(),
            decryptResponse.plaintext().asByteArray()
        );
        
        return new String(plaintext, StandardCharsets.UTF_8);
    }
    
    // Data key caching for performance
    public DataKey getCachedDataKey(String context) {
        return dataKeyCache.computeIfAbsent(context, ctx -> {
            GenerateDataKeyResponse response = kmsClient.generateDataKey(
                GenerateDataKeyRequest.builder()
                    .keyId(keyId)
                    .keySpec("AES_256")
                    .encryptionContext(Map.of("purpose", ctx))
                    .build()
            );
            
            return new DataKey(
                response.plaintext().asByteArray(),
                response.ciphertextBlob().asByteArray()
            );
        });
    }
}
```

## Certificate Lifecycle Management

```java
@Service
public class CertificateManager {
    
    private final Map<String, X509Certificate> certificateCache = new ConcurrentHashMap<>();
    private final ScheduledExecutorService renewer = Executors.newScheduledThreadPool(1);
    
    @PostConstruct
    public void startCertificateRenewal() {
        // Check certificates daily and renew those expiring within 30 days
        renewer.scheduleAtFixedRate(this::renewExpiringCertificates, 
            0, 1, TimeUnit.DAYS);
    }
    
    public X509Certificate getCertificate(String domain) {
        return certificateCache.computeIfAbsent(domain, d -> {
            try {
                KeyStore keyStore = KeyStore.getInstance("PKCS12");
                keyStore.load(new FileInputStream(d + ".p12"), 
                    getPassword(d).toCharArray());
                
                String alias = keyStore.aliases().nextElement();
                return (X509Certificate) keyStore.getCertificate(alias);
            } catch (Exception e) {
                throw new CertificateException("Failed to load certificate", e);
            }
        });
    }
    
    private void renewExpiringCertificates() {
        for (Map.Entry<String, X509Certificate> entry : certificateCache.entrySet()) {
            try {
                Date notAfter = entry.getValue().getNotAfter();
                Date renewalThreshold = Date.from(
                    Instant.now().plus(30, ChronoUnit.DAYS));
                
                if (notAfter.before(renewalThreshold)) {
                    log.info("Renewing certificate for {} (expires {})", 
                        entry.getKey(), notAfter);
                    
                    // Request new certificate from CA (e.g., ACM or Let's Encrypt)
                    X509Certificate newCert = requestCertificate(entry.getKey());
                    
                    // Update keystore
                    updateKeyStore(entry.getKey(), newCert);
                    
                    // Update runtime cache
                    certificateCache.put(entry.getKey(), newCert);
                    
                    log.info("Certificate renewed for {}", entry.getKey());
                }
            } catch (Exception e) {
                log.error("Failed to renew certificate for {}", entry.getKey(), e);
            }
        }
    }
}
```

## Encryption at Rest: Database Level

```java
@Entity
@Table(name = "sensitive_data")
public class SensitiveEntity {
    
    @Id
    private Long id;
    
    @Convert(converter = EncryptionConverter.class)
    @Column(name = "encrypted_field")
    private String sensitiveField;
}

@Converter
public class EncryptionConverter implements AttributeConverter<String, String> {
    
    private final EncryptionService encryptionService;
    
    public EncryptionConverter() {
        this.encryptionService = ApplicationContextProvider.getBean(EncryptionService.class);
    }
    
    @Override
    public String convertToDatabaseColumn(String attribute) {
        if (attribute == null) return null;
        return encryptionService.encrypt(attribute);
    }
    
    @Override
    public String convertToEntityAttribute(String dbData) {
        if (dbData == null) return null;
        return encryptionService.decrypt(dbData);
    }
}
```

## Best Practices

- Use TLS 1.3 exclusively; disable TLS 1.2 and below to avoid downgrade attacks and weak cipher suites
- Implement mTLS for all service-to-service communication in zero-trust architectures (service mesh sidecars handle this transparently)
- Use envelope encryption (KMS with data keys) rather than encrypting large payloads directly with a master key
- Rotate encryption keys regularly; automate key rotation using AWS KMS key rotation or HashiCorp Vault
- Store secrets and private keys in a secrets manager (AWS Secrets Manager, Vault, Kubernetes Secrets), never in code or config files
- Use short-lived certificates (e.g., 90-day Let's Encrypt) and automate renewal to reduce the blast radius of compromised keys
- Implement HSM-backed key storage for regulatory compliance (PCI-DSS, HIPAA) to ensure keys never leave hardware

## Common Mistakes

- Using self-signed certificates without proper CA management, causing trust failures in production service mesh configurations
- Storing encryption keys alongside encrypted data in the same database, defeating the purpose of encryption
- Not encrypting backups or database snapshots, leaving data exposed via backup storage
- Using ECB mode for AES encryption, which reveals data patterns (always use GCM or CBC with proper IV)
- Setting excessively long TLS session ticket lifetimes, increasing the risk of session hijacking
- Hard-coding certificate passwords or private keys in source code or environment variables
- Not configuring certificate revocation (CRL/OCSP), allowing compromised certificates to remain trusted indefinitely

## Summary

Encryption protects data in transit through TLS 1.3 and mTLS, and at rest through envelope encryption with KMS. TLS 1.3's 1-RTT handshake and forward secrecy provide strong transport security with minimal latency overhead. For data at rest, envelope encryption combines the performance of symmetric encryption with the manageability of asymmetric key wrapping. The operational foundation of any encryption strategy is automated certificate lifecycle management, key rotation, and strict separation of key material from encrypted data.
