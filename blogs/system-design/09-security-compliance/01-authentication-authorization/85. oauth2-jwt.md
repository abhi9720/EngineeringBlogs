# OAuth 2.0 & JWT Authentication

## Overview

OAuth 2.0 is the industry-standard protocol for authorization, enabling applications to obtain limited access to user accounts. JSON Web Tokens (JWT) provide a compact, self-contained format for transmitting claims between parties. Together, they form the backbone of modern API security.

## JWT Structure

A JWT consists of three Base64-encoded parts separated by dots: Header, Payload, and Signature.

```java
// JWT Structure: header.payload.signature
// Example:
// eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9.
// eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkFiIiwiaWF0IjoxNTE2MjM5MDIyfQ.
// SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c

public class JwtStructure {
    public JwtParts parse(String jwt) {
        String[] parts = jwt.split("\\.");
        if (parts.length != 3) {
            throw new IllegalArgumentException("Invalid JWT format");
        }
        
        return JwtParts.builder()
            .header(decodeHeader(parts[0]))
            .payload(decodePayload(parts[1]))
            .signature(parts[2])
            .build();
    }
    
    private JwtHeader decodeHeader(String encoded) {
        String json = new String(Base64.getUrlDecoder().decode(encoded));
        return objectMapper.readValue(json, JwtHeader.class);
        // {"alg": "RS256", "typ": "JWT", "kid": "key-id-1"}
    }
    
    private JwtPayload decodePayload(String encoded) {
        String json = new String(Base64.getUrlDecoder().decode(encoded));
        return objectMapper.readValue(json, JwtPayload.class);
        // {"sub": "123", "iss": "https://auth.example.com",
        //  "aud": ["api.example.com"], "exp": 1715000000,
        //  "iat": 1714913600, "scope": "read write"}
    }
}
```

## Spring Security OAuth 2.0 Configuration

### Authorization Server Configuration

```java
@Configuration
@EnableAuthorizationServer
public class AuthorizationServerConfig {
    
    @Bean
    public JwtAccessTokenConverter accessTokenConverter() {
        JwtAccessTokenConverter converter = new JwtAccessTokenConverter();
        converter.setSigningKey(RsaKeyUtils.generateRsaKeyPair());
        converter.setVerifierKey(publicKey);
        return converter;
    }
    
    @Bean
    public AuthorizationCodeServices authorizationCodeServices() {
        return new InMemoryAuthorizationCodeServices();
    }
    
    @Bean
    public TokenStore tokenStore() {
        return new JwtTokenStore(accessTokenConverter());
    }
}
```

### Resource Server Configuration

```java
@Configuration
@EnableResourceServer
@EnableGlobalMethodSecurity(prePostEnabled = true)
public class ResourceServerConfig extends ResourceServerConfigurerAdapter {
    
    @Override
    public void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/api/public/**").permitAll()
                .antMatchers("/api/admin/**").hasAuthority("SCOPE_admin")
                .antMatchers("/api/**").authenticated()
            .and()
            .oauth2ResourceServer()
                .jwt()
                .jwtAuthenticationConverter(jwtAuthenticationConverter());
    }
    
    @Bean
    public JwtDecoder jwtDecoder() {
        return NimbusJwtDecoder.withJwkSetUri("https://auth.example.com/.well-known/jwks.json")
            .jwsAlgorithm(SignatureAlgorithm.RS256)
            .build();
    }
    
    private Converter<Jwt, AbstractAuthenticationToken> jwtAuthenticationConverter() {
        JwtAuthenticationConverter converter = new JwtAuthenticationConverter();
        converter.setJwtGrantedAuthoritiesConverter(jwt -> {
            Collection<String> scopes = jwt.getClaimAsStringList("scope");
            return scopes.stream()
                .map(s -> new SimpleGrantedAuthority("SCOPE_" + s))
                .collect(Collectors.toList());
        });
        return converter;
    }
}
```

## OAuth 2.0 Flows in Practice

### Authorization Code Flow with PKCE

```java
@Service
public class OAuthClientService {
    
    private final WebClient webClient;
    
    public String initiateLogin(String clientId, String redirectUri) {
        // Generate PKCE code challenge
        SecureRandom secureRandom = new SecureRandom();
        byte[] codeVerifier = new byte[32];
        secureRandom.nextBytes(codeVerifier);
        String verifier = Base64.getUrlEncoder().withoutPadding()
            .encodeToString(codeVerifier);
        
        // Store verifier for the session (in production, use session store)
        sessionStore.put("code_verifier", verifier);
        
        // Compute code challenge (S256)
        String challenge = generateCodeChallenge(verifier);
        
        // Build authorization URL
        return UriComponentsBuilder
            .fromUriString("https://auth.example.com/oauth2/authorize")
            .queryParam("response_type", "code")
            .queryParam("client_id", clientId)
            .queryParam("redirect_uri", redirectUri)
            .queryParam("code_challenge", challenge)
            .queryParam("code_challenge_method", "S256")
            .queryParam("scope", "openid profile email")
            .queryParam("state", generateState())
            .build()
            .toUriString();
    }
    
    public TokenResponse exchangeCodeForToken(String code, String redirectUri) {
        String verifier = sessionStore.get("code_verifier");
        
        return webClient.post()
            .uri("https://auth.example.com/oauth2/token")
            .body(BodyInserters.fromFormData("grant_type", "authorization_code")
                .with("code", code)
                .with("redirect_uri", redirectUri)
                .with("client_id", clientConfig.getClientId())
                .with("code_verifier", verifier))
            .retrieve()
            .bodyToMono(TokenResponse.class)
            .block();
    }
    
    private String generateCodeChallenge(String verifier) {
        try {
            MessageDigest md = MessageDigest.getInstance("SHA-256");
            byte[] digest = md.digest(verifier.getBytes(StandardCharsets.US_ASCII));
            return Base64.getUrlEncoder().withoutPadding().encodeToString(digest);
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException(e);
        }
    }
}
```

### Client Credentials Flow (Machine-to-Machine)

```java
@Service
public class MachineToMachineAuth {
    
    private final WebClient webClient;
    private volatile TokenResponse cachedToken;
    private final Object tokenLock = new Object();
    
    public String getAccessToken() {
        if (cachedToken != null && !cachedToken.isExpired()) {
            return cachedToken.getAccessToken();
        }
        
        synchronized (tokenLock) {
            // Double-check after acquiring lock
            if (cachedToken != null && !cachedToken.isExpired()) {
                return cachedToken.getAccessToken();
            }
            
            TokenResponse response = webClient.post()
                .uri("https://auth.example.com/oauth2/token")
                .header("Authorization", basicAuth())
                .body(BodyInserters.fromFormData("grant_type", "client_credentials")
                    .with("scope", "api.read api.write"))
                .retrieve()
                .bodyToMono(TokenResponse.class)
                .block();
            
            cachedToken = response;
            return response.getAccessToken();
        }
    }
    
    private String basicAuth() {
        String credentials = clientId + ":" + clientSecret;
        return "Basic " + Base64.getEncoder().encodeToString(credentials.getBytes());
    }
}
```

## JWT Validation and Verification

```java
@Service
public class JwtValidationService {
    
    private final JwkSetCache jwkSetCache;
    
    public Claims validateToken(String jwt) {
        try {
            // 1. Parse the JWT
            SignedJWT signedJWT = SignedJWT.parse(jwt);
            
            // 2. Verify signature using JWKS
            JWKMatcher matcher = new JWKMatcher.Builder()
                .keyID(signedJWT.getHeader().getKeyID())
                .build();
            
            List<JWK> matches = jwkSetCache.get().getJWKs(matcher);
            if (matches.isEmpty()) {
                throw new InvalidJwtException("No matching JWK found");
            }
            
            JWK jwk = matches.get(0);
            JWSVerifier verifier = new RSASSAVerifier((RSAKey) jwk);
            
            if (!signedJWT.verify(verifier)) {
                throw new InvalidJwtException("Signature verification failed");
            }
            
            // 3. Validate claims
            JWTClaimsSet claims = signedJWT.getJWTClaimsSet();
            
            validateExpiration(claims);
            validateIssuer(claims);
            validateAudience(claims);
            validateNotBefore(claims);
            
            return claims;
            
        } catch (ParseException | JOSEException e) {
            throw new InvalidJwtException("JWT validation failed", e);
        }
    }
    
    private void validateExpiration(JWTClaimsSet claims) {
        Date expiration = claims.getExpirationTime();
        if (expiration != null && expiration.before(new Date())) {
            throw new JwtExpiredException("Token expired at " + expiration);
        }
    }
    
    private void validateIssuer(JWTClaimsSet claims) {
        String issuer = claims.getIssuer();
        if (!expectedIssuers.contains(issuer)) {
            throw new InvalidJwtException("Unknown issuer: " + issuer);
        }
    }
    
    private void validateAudience(JWTClaimsSet claims) {
        List<String> audiences = claims.getAudience();
        if (audiences == null || !audiences.contains(expectedAudience)) {
            throw new InvalidJwtException("Invalid audience");
        }
    }
}
```

## Refresh Token Rotation

```java
@Service
public class RefreshTokenService {
    
    public TokenResponse rotateRefreshToken(String oldRefreshToken) {
        // Validate old refresh token
        RefreshToken stored = tokenRepository.findByToken(oldRefreshToken)
            .orElseThrow(() -> new InvalidTokenException("Refresh token not found"));
        
        if (stored.isUsed()) {
            // Token reuse detected: revoke all tokens for this user
            tokenRepository.revokeAllForUser(stored.getUserId());
            throw new TokenReuseException("Refresh token reuse detected");
        }
        
        // Mark old token as used
        tokenRepository.markUsed(stored.getId());
        
        // Generate new tokens
        String newAccessToken = generateAccessToken(stored.getUserId());
        String newRefreshToken = generateRefreshToken();
        
        // Store new refresh token
        tokenRepository.save(RefreshToken.builder()
            .token(hashToken(newRefreshToken))
            .userId(stored.getUserId())
            .expiresAt(Instant.now().plus(30, ChronoUnit.DAYS))
            .family(stored.getFamily())
            .build());
        
        return new TokenResponse(newAccessToken, newRefreshToken);
    }
}
```

## Best Practices

- Always use PKCE (Proof Key for Code Exchange) even for server-side apps; it defends against authorization code interception
- Keep access tokens short-lived (15-30 minutes) and use refresh tokens for session extension
- Store refresh tokens in an HTTP-only, Secure, SameSite=Strict cookie for browser applications
- Rotate refresh tokens on each use to detect token theft (refresh token rotation pattern)
- Use asymmetric signing (RS256 or ES256) for JWTs so resource servers can verify without sharing secrets
- Validate all JWT claims: exp, nbf, iss, aud, and custom claims like azp (authorized party)
- Implement token revocation via a deny list for short-lived JWTs or a database for refresh tokens

## Common Mistakes

- Storing JWT access tokens in localStorage (vulnerable to XSS; use httpOnly cookies instead)
- Not implementing refresh token rotation, allowing stolen refresh tokens to be used indefinitely
- Using symmetric signing (HS256) across services, requiring secret sharing that increases attack surface
- Accepting JWTs without validating the issuer and audience, allowing tokens from unknown sources
- Setting excessively long token expiration (days or weeks), increasing the window of compromise
- Not validating the token's `nbf` (not before) claim, accepting tokens that are not yet valid
- Exposing client secrets in mobile or SPAs; use PKCE and the authorization code flow instead of implicit flow

## Summary

OAuth 2.0 and JWT form a powerful combination for securing modern applications. The authorization code flow with PKCE provides strong security for both server-side and client-side applications, while JWT enables stateless, self-contained token validation. Spring Security's OAuth 2.0 support simplifies implementation, but careful attention to token lifetimes, refresh rotation, claim validation, and secure storage is essential for production-grade security.
