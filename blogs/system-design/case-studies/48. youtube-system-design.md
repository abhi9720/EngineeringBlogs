# YouTube System Design

## Overview

YouTube serves over 2 billion users, handling 500+ hours of video uploads every minute and delivering billions of hours of content daily. The platform manages the complete video lifecycle—from upload through transcoding, storage, CDN distribution, and adaptive streaming. This case study explores the architectural decisions that make this possible.

This guide covers video upload and processing pipeline, video storage and transcoding, CDN architecture, streaming protocols, recommendation systems, and the techniques enabling massive scale.

## Requirements

### Functional Requirements

1. **Video Upload**: Support upload from mobile, web, API
2. **Transcoding**: Convert to multiple resolutions and formats
3. **Streaming**: Adaptive bitrate streaming
4. **Playback**: Web, mobile, TV, set-top boxes
5. **Recommendations**: Personalized content suggestions
6. **Search**: Full-text video search
7. **Analytics**: View counts, engagement metrics
8. **Live Streaming**: Real-time broadcast capability

### Non-Functional Requirements

| Requirement | Target |
|---|---|
| Upload speed | Real-time or faster |
| Processing time | < 10 minutes for typical video |
| Start latency | < 2 seconds |
| Availability | 99.9% |
| Buffer time | < 3 seconds |

### Scale Estimates

| Metric | Value |
|---|---|
| Uploads/minute | 500+ hours |
| Daily views | 1B+ |
| Peak QPS | 10M+ |
| Storage | Exabytes |
| CDN edge locations | 200+ |

## Video Upload Pipeline

### Chunked Upload

For reliability, videos are uploaded in chunks:

```java
@Service
public class ChunkedUploadService {
    
    public UploadSession createSession(UploadRequest request) {
        String uploadId = generateUploadId();
        
        UploadSession session = UploadSession.builder()
            .id(uploadId)
            .videoId(request.getVideoId())
            .totalChunks(request.getTotalChunks())
            .status(UploadStatus.IN_PROGRESS)
            .build();
        
        return uploadSessionRepository.save(session);
    }
    
    @Transactional
    public ChunkResult uploadChunk(ChunkUploadRequest request) {
        // Get or create session
        UploadSession session = getSession(request.getUploadId());
        
        // Append chunk to storage
        String chunkPath = saveChunk(
            session.getVideoId(),
            request.getChunkIndex(),
            request.getData()
        );
        
        // Update progress
        session.addChunk(request.getChunkIndex());
        
        // Check if complete
        if (session.isComplete()) {
            initiateTranscoding(session.getVideoId());
        }
        
        return ChunkResult.success(session.getProgress());
    }
}
```

## Video Transcoding Pipeline

### Why Transcoding?

Raw uploads are transcoded into multiple formats for:
- **Resolution**: 144p to 4K+
- **Format**: Different codec compatibility
- **Quality**: Multiple quality levels
- **Adaptive Streaming**: HLS/DASH segments

### Transcoding Pipeline

```java
@Service
public class TranscodingPipeline {
    
    public void initiateTranscoding(String videoId) {
        // Get video metadata
        Video video = videoRepository.findById(videoId);
        
        // Determine output formats
        List<OutputFormat> outputs = determineOutputs(video.getResolution());
        
        // Create transcoding jobs
        for (OutputFormat output : outputs) {
            TranscodeJob job = TranscodeJob.builder()
                .videoId(videoId)
                .outputFormat(output)
                .status(JobStatus.QUEUED)
                .build();
            
            transcodingQueue.send(job);
        }
    }
}

@Service
public class TranscoderWorker {
    
    public void processJob(TranscodeJob job) {
        // Input: Uploaded video
        // Output: Multiple quality variants
        
        // Step 1: Decode
        MediaDecoder decoder = createDecoder(job.getInputPath());
        
        // Step 2: Transcode each quality
        for (Quality quality : job.getOutputFormats()) {
            MediaEncoder encoder = createEncoder(quality);
            encode(decoder, encoder, job.getOutputPath(quality));
        }
        
        // Step 3: Generate thumbnails
        generateThumbnails(job.getInputPath(), job.getVideoId());
        
        // Step 4: Generate manifest
        generateManifest(job.getVideoId());
        
        // Clean up
        cleanup(job.getInputPath());
    }
}
```

## Video Storage

### Object Storage Architecture

```java
@Service
public class VideoStorageService {
    
    public String storeVideo(String videoId, byte[] data, VideoFormat format) {
        String key = buildKey(videoId, format);
        
        // Store in GCS/S3
        storage.putObject(
            bucketName,
            key,
            new ByteArrayInputStream(data),
            buildMetadata(format)
        );
        
        return key;
    }
    
    public String storeSegment(String videoId, int quality, int segmentIndex, byte[] data) {
        String key = String.format("%s/%dp/%03d.ts", videoId, quality, segmentIndex);
        storage.putObject(bucketName, key, data);
        return key;
    }
}
```

## CDN Distribution

### CDN Architecture

YouTube uses Google's private CDN infrastructure:

```java
@Service
public class CdnService {
    
    public String getCdnUrl(VideoSegment segment) {
        // Get fastest edge for user
        String userLocation = getUserLocation();
        EdgeServer edge = edgeSelector.selectBest(userLocation, segment);
        
        return edge.getUrl(segment);
    }
    
    public void cacheVideo(Video video) {
        // Pre-position popular videos
        List<String> popular = analyticsService.getPopularVideos();
        
        for (String videoId : popular) {
            cdnCache.prefetch(videoId);
        }
    }
}
```

### Edge Caching Strategy

```
User → Closest Edge → Cache Hit → Return
                        ↓ Cache Miss
                 Origin Shield
                        ↓
                  Origin Storage
```

### Cache Hierarchy

1. **Browser Cache**: Session-based
2. **ISP Cache**: Local ISP caching
3. **Edge CDN**: 200+ locations
4. **Origin Shield**: Regional caches
5. **Origin Storage**: Final backend

HLS splits videos into small segments:

### DASH (Dynamic Adaptive Streaming)

DASH provides more control:


### Player Implementation

```java
public class AdaptivePlayer {
    
    public void selectQuality() {
        // Monitor buffer level
        double bufferLevel = getBufferLevel();
        
        // Monitor network throughput
        double throughput = measureThroughput();
        
        // Calculate optimal quality
        int targetBitrate = calculateBitrate(bufferLevel, throughput);
        
        // Switch if needed
        if (shouldSwitch(targetBitrate)) {
            switchQuality(targetBitrate);
        }
    }
    
    private int calculateBitrate(double bufferLevel, double throughput) {
        // Conservative: Use 50% of throughput
        // But don't drain buffer过快
        if (bufferLevel < 5) {
            // Emergency: Lower quality
            return Math.max(360, currentBitrate / 2);
        } else if (bufferLevel < 10) {
            // Normal: 40% of throughput
            return (int) (throughput * 0.4);
        } else {
            // Aggressive: 60% of throughput
            return (int) (throughput * 0.6);
        }
    }
}
```

```java
public class RecommendationService {
    
    public List<Video> getRecommendations(String userId, int limit) {
        // Get user's history
        List<String> watchHistory = historyService.getHistory(userId);
        
        // Get co-watch similarity
        List<String> similarUsers = similarityService.findSimilar(watchHistory);
        
        // Get videos from similar users
        List<Video> collaborative = getVideosFromSimilarUsers(similarUsers);

        List<Video> contentBased = contentRecommender.recommend(userId, limit);

        // Merge and rank
        return mergeAndRank(collaborative, contentBased, trending);
    }
}
```

## Best Practices

1. **Multi-quality encoding**: Support all device capabilities
2. **CDN pre-positioning**: Cache popular content before demand
3. **Adaptive bitrate**: Match quality to network conditions
4. **Chunked uploads**: Handle unreliable uploads
5. **Parallel transcode**: Process multiple qualities concurrently
6. **Manifest optimization**: Reduce manifest sizes
7. **Buffer management**: Pre-buffer for smooth playback

## Common Mistakes

1. **Single quality**: Not supporting all devices
2. **No CDN**: Direct storage fetching causes latency
3. **No transcoding**: Upload format may not play
4. **Large segments**: Causes rebuffering
5. **No adaptive streaming**: Quality can't adapt

## Summary

YouTube's architecture demonstrates production-grade video handling:

1. **Upload pipeline**: Chunked uploads with resumability
2. **Transcoding**: Multiple quality outputs in parallel
3. **CDN**: Global edge network for low latency
4. **Adaptive streaming**: HLS/DASH for quality adaptation
5. **Recommendations**: Collaborative + content-based ranking

The combination enables billions of daily views with sub-second playback starts.
