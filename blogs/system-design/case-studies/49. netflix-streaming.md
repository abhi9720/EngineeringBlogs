# Netflix Streaming System Design

## Overview

Netflix serves over 200 million subscribers across 190 countries, delivering streaming content to every internet-enabled screen. The platform processes an estimated 15% of global internet bandwidth during peak hours. This case study explores Netflix's architectural decisions that enable seamless streaming to millions of concurrent viewers with personalization at scale.

This guide covers content delivery networks, encoding strategies, recommendation engines, device profiles, and the operational excellence that powers Netflix's streaming experience.

## Requirements

### Functional Requirements

1. **Universal Playback**: Support thousands of device types
2. **Instant Start**: Begin playback within 2 seconds
3. **Adaptive Streaming**: Automatic quality adjustment
4. **Personalization**: Home screen tailored to each user
5. **Continue Watching**: Resume from last position
6. **Offline Viewing**: Download for offline play
7. **Multiple Profiles**: Separate recommendations per profile
8. **Parental Controls**: Content restrictions

### Non-Functional Requirements

| Requirement | Target |
|---|---|
| Start latency | < 2 seconds |
| Availability | 99.9% |
| Error rate | < 0.1% |
| Buffer length | > 7-10 seconds |
| Quality switches | Seamless |

### Scale Estimates

| Metric | Value |
|---|---|
| Subscribers | 200M+ |
| Countries | 190 |
| Streaming hours/day | 150M+ |
| Catalog size | 15,000+ titles |

```java
@Service
public class EncodingPipeline {
    
    public List<VideoVariant> encodeVideo(Video source) {
        List<VideoVariant> variants = new ArrayList<>();
        
        // H.264 variants
        variants.addAll(encodeH264(source, Arrays.asList(
            new Quality(640, 360, "SD"),
            new Quality(1280, 720, "HD"),
            new Quality(1920, 1080, "FHD")
        )));
        
        // VP9 variants
        if (supportsVp9(getDevices())) {
            variants.addAll(encodeVp9(source, Arrays.asList(
                new Quality(1280, 720, "HD"),
                new Quality(1920, 1080, "FHD"),
                new Quality(3840, 2160, "4K")
            )));
        }
        
        return variants;
    }
}
```

### Quality Switching Algorithm

```java
public class AdaptiveQualitySelector {
    
    public int selectQuality(PlayerState state) {
        double bufferLevel = state.getBufferSeconds();
        double bandwidth = state.getEstimateBandwidth();
        double currentQuality = state.getCurrentQuality();
        
        // Conservative switch-down
        if (bufferLevel < 3) {
            return Math.max(0, currentQuality - 1);
        }
        
        // Aggressive switch-up with delay
        if (bufferLevel > 15 && bandwidth > currentQuality * 1.5) {
            return Math.min(maxQuality, currentQuality + 1);
        }
        
        return currentQuality;
    }
}
```

###Bandwidth Estimation

```java
public class BandwidthEstimator {
    
    public double estimateBandwidth(DownloadHistory history) {
        // Exponential moving average of recent downloads
        double recentWeight = 0.7;
        double olderWeight = 0.3;
        
        List<DownloadSample> samples = history.getRecentSamples(10);
        
        double totalBandwidth = 0;
        double totalWeight = 0;
        
        for (int i = 0; i < samples.size(); i++) {
            double weight = Math.pow(i / samples.size(), 1/3);
            totalBandwidth += samples.get(i).getBandwidth() * weight;
            totalWeight += weight;
        }
        
        return totalBandwidth / totalWeight;
    }
}
```

### Buffer Management

```java
public class BufferManager {
    
    public void onDownloadComplete(DownloadSegment segment) {
        // Add to buffer
        buffer.add(segment);
        
        // Update metrics
        metrics.recordBufferLevel(buffer.size());
        
        // Trigger playback if ready
        if (shouldStartPlayback()) {
            player.start();
        }
    }
    
    private boolean shouldStartPlayback() {
        // Start if buffer has minimum content
        return buffer.getDurationSeconds() >= MIN_START_DURATION &&
               !player.isPlaying();
    }
}
```

### Ranking Model

```python
# Simplified ranking model
def rank_title(title, user_features):
    score = 0
    
    # Relevance to viewing history (40%)
    score += 0.40 * cosine_similarity(
        user_embedding, 
        title_embedding
    )
    
    # Freshness (20%)
    score += 0.20 * freshness_score(title.release_date)
    
    # Popularity in user's region (15%)
    score += 0.15 * regional_popularity(title)
    
    # Match quality signals (15%)
    score += 0.15 * quality_score(title.rating, title.metadata)
    
    # Diversity penalty for category repeats (10%)
    score += 0.10 * diversity_score(user_recent_cats, title.category)
    
    return score
```

### Personalization at Scale

```java
@Service
public class PersonalizedHomeService {
    
    public HomeScreen getHomeScreen(String userId, Profile profile) {
        // Build personalized rows in parallel
        
        // "Continue Watching" - async
        CompletableFuture<Row> continueWatching = 
            fetchContinueWatching(userId, profile);
        
        // "Because you watched X" - async  
        CompletableFuture<List<Row>> becauseYouWatched = 
            fetchBecauseYouWatched(userId, profile);
        
        // "Trending Now" - shared across users
        CompletableFuture<Row> trending = 
            fetchTrending(profile.getRegion());
        
        // "Top Picks" - personalized
        CompletableFuture<Row> topPicks = 
            fetchTopPicks(userId, profile);
        
        // Wait for all and combine
        List<Row> rows = Stream.of(
                continueWatching, becauseYouWatched, trending, topPicks
            )
            .map(CompletableFuture::join)
            .filter(Objects::nonNull)
            .collect(Collectors.toList());
        
        return new HomeScreen(rows);
    }
}
```

## Search

### Search Implementation

```java
@Service
public class SearchService {
    
    public SearchResults search(String query, SearchOptions options) {
        // Build search query
        BoolQuery.Builder queryBuilder = new BoolQuery.Builder();
        
        // Title match
        queryBuilder.must(QueryBuilders.matchQuery("title", query));
        
        // Boost exact matches
        queryBuilder.should(QueryBuilders.matchQuery("title.exact", query)
            .boost(10f));
        
        // Filter by enabled profiles
        for (Profile profile : options.getProfiles()) {
            queryBuilder.filter(buildProfileFilter(profile));
        }
        
        // Execute
        SearchHits<Video> hits = elasticsearch.search(queryBuilder.build());
        
        return toSearchResults(hits);
    }
}
```

## Best Practices

1. **Custom CDN**: Open Connect places content at ISPs for minimal latency.
2. **Multiple codecs**: Support all device types with optimal encoding.
3. **Adaptive quality**: Seamlessly adjust to network conditions.
4. **Aggressive caching**: Buffer 7-10 seconds for smooth playback.
5. **Personalization**: Tailor home screen for each user and profile.
6. **Regional encoding**: Match content to regional licensability.
7. **Monitor quality**: Track rebuffering and quality changes.

## Common Mistakes

1. **Single quality only**: Not supporting all devices.
2. **No CDN**: Direct origin serving causes high latency.
3. **Small buffer**: Leads to rebuffering.
4. **No personalization**: Generic home screen.
5. **Ignoring bandwidth**: Quality doesn't adapt.

## Summary

Netflix's streaming architecture demonstrates:

1. **Custom CDN**: Open Connect places servers at ISPs globally.
2. **Multi-format encoding**: Support every device capability.
3. **Adaptive streaming**: DASH with quality switching.
4. **Personalized ranking**: ML models for each user.
5. **Aggressive buffering**: 7-10+ seconds for smooth playback.

This architecture enables Netflix to deliver 150M+ streaming hours daily with 99.9% availability.
