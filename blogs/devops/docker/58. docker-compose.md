# Docker Compose

## Overview

Docker Compose is a tool for defining and running multi-container Docker applications. With Compose, you use a YAML file to configure your application's services.

---

## Problem Statement

How do you manage multi-container applications where each service has its own configuration? Docker Compose simplifies this complexity.

---

## Main Content Section 1

### Docker Compose Basics

Docker Compose allows you to:
* Define services in a single docker-compose.yml file
* Start all services with a single command
* Scale services up and down

---

## Main Content Section 2

### Key Concepts

* **Services**: Containers in your application
* **Volumes**: Persistent data storage
* **Networks**: Communication between services

---

## Code Example

```yaml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "5000:5000"
    environment:
      - DATABASE_URL=postgres://db:5432/mydb
    depends_on:
      - db

  db:
    image: postgres:15
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_PASSWORD=secret

volumes:
  postgres_data:
```

---

## Best Practices

* Use environment variables for configuration
* Define healthchecks for dependencies
* Use named volumes for persistent data
* Keep your compose file versioned

---

## Common Mistakes

* Not specifying restart policies
* Forgetting to expose required ports
* Not setting proper resource limits
* Ignoring dependency ordering

---

## Summary

Docker Compose streamlines managing multi-container applications, making it essential for local development and testing environments.
