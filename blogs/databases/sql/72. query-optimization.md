# Query Optimization

## Overview

Query optimization is the process of improving SQL query performance to reduce execution time and resource consumption. A well-optimized query can be hundreds or thousands of times faster than a poorly written one.

---

## Problem Statement

Slow queries can bottleneck entire applications, leading to poor user experience, increased server load, and higher infrastructure costs. Identifying and optimizing slow queries is critical for maintaining system performance.

---

## Main Content Section 1

### Identifying Slow Queries

* Use EXPLAIN/EXPLAIN ANALYZE to view query execution plans
* Enable query logging for production monitoring
* Use database profiling tools
* Monitor slow query logs (typically queries > 1 second)

---

## Main Content Section 2

### Optimization Techniques

* Select only required columns (avoid SELECT *)
* Use appropriate indexes on WHERE and JOIN columns
* Avoid functions on indexed columns in WHERE clauses
* Use LIMIT for pagination
* Avoid subqueries when JOINs suffice

---

## Code Example

```sql
-- Use EXPLAIN to analyze query plan
EXPLAIN ANALYZE
SELECT u.name, COUNT(o.id) as order_count
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.created_at > '2024-01-01'
GROUP BY u.name
HAVING COUNT(o.id) > 5;

-- Instead of this:
SELECT * FROM users; -- Avoid SELECT *

-- Use indexed columns properly
WHERE YEAR(created_at) = 2024; -- Bad: function on column
WHERE created_at >= '2024-01-01' AND created_at < '2025-01-01'; -- Good
```

---

## Best Practices

* Create covering indexes for frequently used queries
* Batch multiple INSERT/UPDATE operations
* Use appropriate data types (avoid string storage for dates)
* Denormalize strategically for read-heavy workloads
* Partition large tables by date or range

---

## Common Mistakes

* Using SELECT * unnecessarily
* Not analyzing execution plans
* Missing indexes on foreign keys
* Using LIKE patterns that prevent index usage
* Over-normalizing for complex reporting queries

---

## Summary

Query optimization requires understanding both your data and your database's execution engine. Regular monitoring and analysis of query performance helps maintain optimal application responsiveness.
