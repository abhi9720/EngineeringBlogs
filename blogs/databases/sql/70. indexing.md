# Indexing in Databases

## Overview

Database indexes are data structures that improve the speed of data retrieval operations on a database table. They work similarly to a book's index, allowing the database to find data without scanning the entire table.

---

## Problem Statement

Without indexes, queries that filter or sort data require full table scans, which become increasingly slow as data grows. Indexes solve this by providing quick lookup paths to the actual data rows.

---

## Main Content Section 1

### How Indexes Work

* Indexes store a sorted copy of the indexed column values
* Each index entry points to the actual data row location
* The database maintains the index as data changes
* Pointers allow O(log n) lookups instead of O(n) scans

---

## Main Content Section 2

### Types of Indexes

* **B-Tree Indexes**: Most common, good for equality and range queries
* **Hash Indexes**: Fast for exact match lookups
* **Composite Indexes**: Index on multiple columns
* **Unique Indexes**: Enforce uniqueness constraints
* **Full-Text Indexes**: For text search operations

---

## Code Example

```sql
-- Create a basic index
CREATE INDEX idx_user_email ON users(email);

-- Create a composite index
CREATE INDEX idx_orders_customer_date ON orders(customer_id, order_date);

-- Create a unique index
CREATE UNIQUE INDEX idx_user_username ON users(username);

-- View existing indexes
SELECT * FROM pg_indexes WHERE tablename = 'users';
```

---

## Best Practices

* Index columns used in WHERE clauses
* Index columns used in JOIN conditions
* Index columns used in ORDER BY
* Consider column cardinality (high selectivity is better)
* Monitor index usage and remove unused indexes

---

## Common Mistakes

* Indexing every column increases write overhead
* Using indexes on low-cardinality columns provides minimal benefit
* Neglecting to update statistics after index creation
* Creating too many indexes slows down INSERT/UPDATE operations

---

## Summary

Indexes are essential for database performance but must be used strategically. Choose the right index type and columns based on your query patterns while balancing write performance costs.
